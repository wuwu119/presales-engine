# 资质证书 (`knowledge/certs/`)

公司所有公开和内部的资质证书扫描件。**评分杠杆中权重最高的一类——招标文件里所有"硬性资质要求"的证据都在这里**。

## 放什么

- **体系认证**：ISO 9001、ISO 27001、ISO 14001、ISO 20000、CMMI
- **行业许可**：ICP 许可证、电信增值业务经营许可证、软件企业认证、高新技术企业证书
- **安全测评**：等保三级 / 分保测评 / 密评报告
- **财务资质**：营业执照、税务登记、近 3 年审计报告
- **合规**：数据安全评估报告、个保法认证等

## 命名约定（推荐）

`<标准或证书名>-<发证机构>-<有效期到>.pdf`，示例：

- `iso9001-SGS-2027-12-31.pdf`
- `iso27001-DNV-2026-06-30.pdf`
- `等保三级-公安三所-2028-03-15.pdf`
- `高新技术企业-科技部-2026-12-31.pdf`

把到期日放在文件名里便于 `ps:rfp-analyze` 快速识别过期证书。

## 格式

PDF 优先（扫描件或原生）。每个证书**独立一个文件**，不要把多个证书合并成一个 PDF。

## 在 company-profile.yaml 里的引用

```yaml
qualifications:
  - id: QUAL-001
    name: ISO 9001 质量管理体系认证
    issuer: SGS
    valid_from: 2024-01-01
    valid_until: 2027-12-31
    evidence_file: certs/iso9001-SGS-2027-12-31.pdf
```

## 谁引用它

- `ps:rfp-analyze` **废标风险扫描**段：逐条对照 RFP 的 `qualifications` 硬性要求，判断"满足 / 勉强满足 / 硬伤"
- 未来 `ps:knowledge-ingest` 会自动从 PDF 提取有效期生成 `qualifications[]` 条目
