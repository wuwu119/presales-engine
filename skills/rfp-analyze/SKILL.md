# ps:rfp-analyze — 投标战略分析

## 前置条件

- `{slug}/分析/rfp.yaml` 存在且通过校验
- `${PRESALES_HOME}/知识库/company-profile.yaml` 存在（setup 阶段创建）

前置条件不满足时**拒绝执行**，提示用户先跑 `/ps:rfp-parse` 或 `/ps:setup`。

## 输入

| 文件 | 必需 |
|------|------|
| `{slug}/分析/rfp.yaml` | ✅ |
| `知识库/company-profile.yaml` | ✅ |
| `知识库/产品档案/*.yaml` | ❌ 可选，用于产品匹配 |
| `知识库/竞品/*.yaml` | ❌ v0.2 引入 |
| `cases/` 下的历史案例 | ❌ v0.3 自动检索，v0.1 人工引用 |

## 输出

`{slug}/分析/analysis.md`

## 必含段落（按固定顺序）

### 1. 执行摘要
- 一句话结论：`Go` / `No-Go` / `Conditional-Go`
- 3 句话理由（最重要的 3 个决策依据）
- 建议投入工时和人员规模

### 2. 废标风险扫描
逐条对照 `rfp.yaml.qualifications` 和 `rfp.yaml.disqualifications`，分三类标注：

- ❌ **不满足（硬伤）** — 具体条款 + 为什么不满足
- ⚠️ **勉强满足** — 需要的论证材料 + 证据来源
- ✅ **满足** — 一行简述证据

**硬伤 > 0 → 建议 No-Go**，除非有强力变通方案。

### 3. 评分杠杆分析
按 `rfp.yaml.scoring[].weight` 降序列出所有评分项，每项标注：

| 字段 | 说明 |
|------|------|
| 权重 | 从 rfp.yaml |
| 我方优势 | `high` / `medium` / `low` / `none` |
| 关键证据 | 引用 company-profile 或 products 中的具体条目 |
| 提分空间 | 可主动加强的部分 |

**高权重（≥ 15%）+ high/medium 优势** = 评分杠杆，列入标书重点章节。

### 4. 竞品格局推断
基于 RFP 语气、评分项偏好、资质门槛做**定性推断**：

- 可能的陪标对象（哪几家常出现在类似项目）
- 甲方倾向的技术路线（从评分项偏好推断）
- 是否疑似定标（硬性要求只有某家能满足）
- 价格锚定区间（基于项目规模和行业惯例）

v0.1 做定性判断，不做定量分析。

### 5. Go/No-Go 决策矩阵

| 维度 | 评分（1-5） | 说明 |
|------|-------------|------|
| 废标风险 | - | 5 = 无硬伤 |
| 评分匹配度 | - | 5 = 高权重项全部有优势 |
| 金额吸引力 | - | 5 = 显著高于团队日常单 |
| 战略价值 | - | 5 = 标杆客户 / 新行业入场 |
| 交付风险 | - | 5 = 技术成熟、交付可控 |
| 竞争激烈度 | - | 5 = 竞争温和 |

**决策阈值**：
- 总分 > 18 → `Go`
- 总分 12-18 → `Conditional-Go`（需补足条件）
- 总分 < 12 → `No-Go`

### 6. 关键信息缺口
列出必须和客户澄清的问题（对应 `rfp.yaml` 中标 `null` 或 `ambiguities` 的字段）。每个问题给出建议沟通方式：

- 📝 正式澄清函（走官方渠道）
- 🤝 销售私下沟通（避免高调）
- 🚫 回避（可能暴露弱点）

### 7. 投标建议

根据决策分叉：

- **Go** → 关键差异化点 / 预估工时 / 建议参与人员 / 需要立即启动的准备工作
- **Conditional-Go** → 需补足的条件清单（按优先级）和 deadline
- **No-Go** → 放弃理由 + 是否值得归档到 `cases/` 作为负向学习样本

## 约束

- **客观中立** — 不迎合销售的"一定要投"，也不过度保守
- **证据驱动** — 每个判断引用 RFP 原文或 `company-profile.yaml` 具体条款
- **禁止编造案例** — 若 `company-profile.yaml` 没有相关案例，直接标"无证据支撑"
- **禁止产品吹嘘** — `products/*.yaml` 中没写的功能不要在分析中"加戏"
- **敏感结论加粗** — `Go/No-Go` 决定必须加粗明显标注

## 失败处理

| 情况 | 处理 |
|------|------|
| `rfp.yaml` 不存在 | 报错 + 指引先跑 `/ps:rfp-parse` |
| `rfp.yaml` 有 `ambiguities` | 继续执行，但在执行摘要标注"⚠️ RFP 解析有歧义，分析结论置信度下降" |
| `company-profile.yaml` 基本信息为空 | 报错 + 提示补全公司档案 |
| 所有评分项我方都 `none` 优势 | 强制 No-Go |

## 后续步骤提示

- **Go / Conditional-Go** → 提示运行 `/ps:bid-draft {slug}`
- **No-Go** → 询问是否立即归档到 `cases/`，记录放弃理由作为负向学习
